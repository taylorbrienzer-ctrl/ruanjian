## 一、参考方向 A：国风山水动态界面

关键词：水墨远山、云雾流动、湖面波纹、留白、金色题签。

优点：

1. 与灵山“山、水、佛、禅、梵宫、湖光”的气质高度一致。
2. 适合做首屏沉浸场景，能降低 AI 数字人的冰冷感。
3. 动态云雾、声波、水纹可以自然承接“语音交互”。

不足：

1. 过度水墨会让科技感不足。
2. 如果只做传统元素，后台数据与 AI 能力不够清晰。

取用策略：

使用远山、湖面、轻雾、金石色强调灵山气质；数字人区域使用柔光和声波动画，让传统美学与 AI 状态连接。

## 二、参考方向 B：智慧文旅/景区数据大屏

关键词：景区地图、路线热力、实时人流、满意度、热点问题、管理看板。

优点：

1. 能直接覆盖赛题中的管理后台、大屏概览、游客洞察。
2. 信息层级清楚，适合评委快速看到完整业务闭环。
3. 能体现景区管理方价值，而不仅是游客端好看。

不足：

1. 常见大屏容易变成深蓝科技模板，与灵山山水气质不匹配。
2. 大量图表如果没有业务语义，会显得空。

取用策略：

保留指标、趋势、热点问答、情绪报告的结构，但把配色改为墨绿、湖青、金石、米白，并用轻量图形模拟趋势。

## 三、参考方向 C：AI 数字人/智能助手界面

关键词：人物舞台、语音波形、文本会话、情绪状态、唇形同步、模型链路。

优点：

1. 直接命中赛题“AI 数字人”的核心记忆点。
2. 适合展示多模态能力：ASR、LLM、RAG、TTS、口型、表情。
3. 游客输入和数字人回答可以做成交互演示。

不足：

1. 通用数字人 UI 容易没有景区行业感。
2. 若人物过大，会挤压路线推荐和景区知识内容。

取用策略：

数字人不做写实视频依赖，采用可托管的 CSS/HTML 舞台表达：轮廓、服饰、声波、表情状态和口型同步指示。视觉上结合灵山山水背景与景区路线信息。

## 四、最终 UI 方案

最终选择“国风山水动态首屏 + 数字人交互中枢 + 文旅管理大屏”的组合。

设计取舍：

1. 首屏优先让评委看见“灵山 AI 数字人导览”，不是普通落地页。
2. 中间核心是数字人和会话，周围放路线推荐、景点知识、模型状态。
3. 后半部分展示管理后台，覆盖知识库、形象管理、感受度报告、数据大屏。
4. 动效集中在云雾、水纹、声波、数字人状态和数据流，避免无意义装饰。
5. 页面保持单文件静态部署，方便 GitHub Pages。
