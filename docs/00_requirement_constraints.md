# 景区导览服务 AI 数字人前端与接口实现约束

依据来源：第十五届中国软件杯 A5《景区导览服务AI数字人》官方赛题页，发布时间 2026-03-23 10:20:11。

## 一、必须踩中的赛题目标

本项目必须呈现一个面向景区的 AI 数字人导览系统，核心目标是让游客通过手机 App 或景区终端与数字人交互，并让管理方通过后台理解游客需求与服务质量。

前端界面必须显性体现：

1. 游客交互端：语音输入、文本输入、数字人语音/表情/口型同步回答的产品形态。
2. 智能问答与讲解：围绕景区历史、文化、景点特色、路线、服务信息提供问答和讲解。
3. 个性化推荐：根据游客兴趣，如历史文化、自然风光、亲子慢游、祈福朝圣，推荐不同路线和讲解重点。
4. 管理后台端：知识库管理、数字人形象管理、游客感受度报告、数据大屏概览。
5. 技术支撑：明确至少一个多模态大模型作为核心 AI 能力，并建立本地景区知识库。
6. 体验指标：景区事实问答准确率目标不低于 90%，语音问答延迟目标小于 5 秒，系统稳定、交互流畅。
7. 可选加分：考虑 GPS 弱定位或室内/山地定位困难场景，提供离线点位、蓝牙信标、视觉地标确认等解决思路。

## 三、前端实现边界

本次交付以前端展示和后端接口设计为主，不实现真实服务端。页面需要通过静态 HTML/CSS/JS 完成可运行演示，适合 GitHub Pages 托管。

必须包含的前端视图：

1. 游客端主界面：灵山山水风格、AI 数字人、语音/文字问答、实时状态、推荐路线。
2. 导览路线视图：多条路线、预计时长、偏好标签、GPS/弱定位提示。
3. 景点讲解视图：景点卡片、文化知识、数字人讲解语气。
4. 管理后台视图：数据大屏、热门问答、满意度、情绪趋势、知识库更新、数字人形象配置。
5. 技术方案视图：接口、模型链路、知识库、响应延迟与准确率目标。

## 四、UI 风格约束

灵山主题应偏“山水、云雾、禅意、金石、湖光”，避免过度赛博、纯深蓝大屏、廉价古风贴纸化。

视觉方向采用三类参考取长补短：

1. 国风山水动态界面：取其墨色层次、远山、雾气、留白和缓慢流动感。
2. 高级智慧文旅大屏：取其数据层级、地图/路线、指标模块和管理后台清晰度。
3. AI 数字人产品界面：取其人物居中、会话浮层、语音波形、情绪状态、唇形/表情提示。

最终页面应做到：

1. 第一屏直接看到“灵山 AI 数字人导览”产品，不做营销空页。
2. 有动态感，但不能遮挡主要内容；动画必须服务于“云、水、声波、数字人状态”。
3. 卡片圆角不超过 8px，避免层层嵌套卡片。
4. 颜色不能单一依赖深蓝或紫色，应以墨绿、湖青、金石、米白、朱砂点缀构成。
5. 移动端和桌面端均不能出现文字溢出、按钮挤压或模块遮挡。

## 五、接口设计约束

后端只做接口设计，接口必须覆盖：

1. 会话与问答：文本问答、语音问答、流式回答、引用知识来源。
2. 多模态数字人：ASR、TTS、口型驱动、表情/情绪状态、数字人形象配置。
3. 路线推荐：游客画像、兴趣、时间、位置、拥挤度、弱定位兜底。
4. 景区知识库：文档上传、切片、向量化、版本管理、问答测试集。
5. 管理分析：游客关注点、情绪趋势、满意度、热门问题、服务建议。
6. 数据大屏：服务人次、响应时延、准确率、知识命中率、异常率。

## 六、演示数据约束

演示内容应优先围绕示范景区资料包中的公开信息组织。若资料包下载失败或解析不足，页面先采用灵山胜境常见公开景点名作为占位演示数据，并在文档中标明需替换为正式知识库切片。
