# App 功能归纳矩阵

## 一、游客端功能

| 功能模块 | 必要功能 | 对应赛题要求 | 前端展示方式 |
| --- | --- | --- | --- |
| AI 数字人首页 | 数字人形象、欢迎语、当前景区状态、语音波形 | 逼真数字人交互、情感互动 | 第一屏数字人舞台、语音状态、情绪标签 |
| 多模态交互 | 语音输入、文本输入、语音输出、表情/口型同步状态 | 支持语音、文本、表情 | 会话输入框、语音按钮、口型同步状态条 |
| 智能问答 | 景区历史、文化、景点特色、服务信息问答 | 精准景区知识问答 | 对话流、知识引用、置信度 |
| 个性化路线 | 按兴趣、时间、体力、人群推荐路线 | 个性化推荐 | 路线卡片、兴趣标签、路线地图 |
| 景点讲解 | 到点自动讲解、深度讲解、儿童/长者语气 | 个性化讲解重点 | 景点卡、讲解模式切换 |
| 情感互动 | 识别游客情绪、鼓励/安抚/提醒 | 情感互动 | 情绪雷达、数字人表情变化 |
| 弱定位兜底 | GPS 弱时使用地标选择、离线点位、蓝牙信标预案 | 可选现实场景加分 | 定位状态提示、重新校准入口 |
| 服务闭环 | 收藏路线、反馈满意度、问题上报 | 游客体验与反馈 | 满意度滑条、反馈标签 |

## 二、管理后台功能

| 功能模块 | 必要功能 | 对应赛题要求 | 前端展示方式 |
| --- | --- | --- | --- |
| 数据大屏 | 今日/本周服务人次、热门问答、满意度趋势 | 数据大屏概览 | 指标条、趋势图、热点问题 |
| 知识库管理 | 上传讲解词、文史资料、FAQ，版本更新 | 知识库管理 | 文档列表、向量化状态、测试通过率 |
| 数字人形象管理 | 外观、服装、声音、语气、表情包配置 | 数字人形象管理 | 形象配置面板 |
| 游客感受度报告 | 关注点分析、情感趋势、服务建议 | 游客感受度报告 | 情绪趋势、关注词云、建议列表 |
| 模型效果监控 | 准确率、响应延迟、知识命中率 | 非功能性需求 | SLA 指标、异常告警 |
| 内容审核 | 低置信度回答复核、敏感问题转人工 | 稳定与准确 | 待复核队列 |

## 三、后端能力分层

| 层级 | 能力 | 推荐技术形态 |
| --- | --- | --- |
| 入口层 | App、景区终端、后台管理台 | REST + SSE/WebSocket |
| AI 编排层 | 意图识别、RAG、工具调用、路线规划 | 多模态大模型 + LangChain/LlamaIndex 类编排 |
| 语音与数字人层 | ASR、TTS、口型同步、表情驱动 | Whisper/ParaTTS/数字人 SDK |
| 知识层 | 文档解析、切片、向量库、标准问答集 | Embedding + 向量数据库 + 文档版本 |
| 数据分析层 | 情绪分析、主题聚类、满意度趋势 | 日志仓库 + 分析任务 |
| 运维层 | 延迟、准确率、异常、审计 | 监控面板 + 告警 |

## 四、演示优先级

| 优先级 | 必做项 | 说明 |
| --- | --- | --- |
| P0 | 游客端数字人问答、路线推荐、后台大屏、知识库管理 | 直接对应 40 分功能完整度 |
| P1 | 多模态链路说明、RAG 准确率目标、口型/表情状态、延迟指标 | 直接对应 30 分技术与创新 |
| P2 | 弱定位、拥挤度避让、游客情绪分析、服务建议 | 增强行业实用性 |
| P3 | 多语言、无障碍、多人同行路线 | 可作为答辩扩展 |
