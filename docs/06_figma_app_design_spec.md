# Figma App 设计稿规格

## 一、文件结构建议

Figma 文件名：`灵山 AI 数字人导览 App`

页面分组：

1. `00 Cover`：产品封面、设计说明。
2. `01 Tokens`：颜色、字体、间距、圆角、阴影。
3. `02 Components`：可复用组件。
4. `03 Visitor App`：游客端核心页面。
5. `04 Admin Console`：管理端移动/平板视图。
6. `05 Prototype`：关键点击流和动效标注。

## 三、组件库

### 1. App 顶部栏

状态：默认、含定位、含返回、含语音状态。

内容：景区名、当前点位、定位精度、天气/人流简短状态。

### 2. 数字人舞台

状态：待机、聆听中、思考中、讲解中、弱网/弱定位。

构成：山水背景、数字人半身、声波条、口型同步点、情绪标签。

### 3. 聊天输入栏

状态：文本输入、语音按住、发送中、禁用。

动作：语音输入、文本发送、快捷问题、清空。

### 4. 路线卡

字段：路线名、时长、适合人群、景点序列、人流状态、推荐理由。

状态：默认、已选中、拥挤提醒、弱定位兜底。

### 5. 景点卡

字段：景点名、文化关键词、讲解摘要、演出/开放时间、收藏状态。

### 6. 数据卡

字段：服务人次、平均响应、知识命中、满意度、热门问答。

用于管理端，不出现在游客首屏。

## 四、游客端画板

### Frame 1：首页导览

尺寸：390 x 844。

布局：

1. 顶部状态栏：灵山胜境、定位“九龙灌浴附近”、人流“适中”。
2. 数字人舞台：占上半屏，展示“灵小山”。
3. 当前回答卡：一句自然讲解 + 知识来源 + 响应时间。
4. 输入栏：语音按钮、文本输入、发送按钮。
5. 快捷问题：亲子半日、祈福拍照、演出时间、弱定位。
6. 底部 Tab：首页、路线、景点、服务、我的。

### Frame 2：路线推荐

布局：

1. 兴趣筛选横向标签。
2. 时间选择：90 分钟、120 分钟、半日、全天。
3. 路线卡列表。
4. 路线详情浮层：景点顺序、推荐理由、拥挤度、开始导览按钮。

### Frame 3：景点讲解

布局：

1. 景点分类：必看、文化、演艺、祈福、休息。
2. 景点卡瀑布/列表。
3. 点击后进入讲解详情：摘要、故事、讲解模式、相关问答。

### Frame 4：行程服务

布局：

1. 定位状态卡。
2. 演出提醒卡。
3. 拥挤度与休息点。
4. 游客反馈入口。

### Frame 5：我的

布局：

1. 偏好：历史文化、自然风光、亲子、祈福。
2. 收藏路线。
3. 历史问答。
4. 声音、语言、无障碍设置。

## 五、管理端画板

建议另建 768 x 1024 平板画板：

1. 运营概览。
2. 热门问答。
3. 知识库维护。
4. 数字人形象管理。
5. 游客感受度报告。

## 六、原型流

1. 首页点击快捷问题 -> 数字人进入“思考中” -> 出现回答卡。
2. 首页点击“生成路线” -> 路线推荐页 -> 选择路线 -> 开始导览。
3. 景点卡点击 -> 景点详情 -> 开始讲解。
4. 定位弱提示 -> 行程服务页 -> 地标确认。
5. 我的偏好修改 -> 回到路线页刷新推荐理由。
